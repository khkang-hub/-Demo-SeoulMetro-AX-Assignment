# 서울교통공사 공식 홈페이지 AX 리뉴얼 프로젝트 리포트

## 1. 프로젝트 개요

본 과제는 서울교통공사 공식 홈페이지를 시민 중심 관점에서 재해석하고, Design Thinking 기반으로 문제를 정의한 뒤, AX 기획서와 PRD를 작성하고, HTML/CSS/JS 기반의 작동 가능한 웹 프로토타입을 제작하는 것을 목표로 한다.

이번 리뉴얼의 핵심 방향은 단순한 시각 개선이 아니라, 시민이 실제로 겪는 문제를 더 빠르게 해결할 수 있는 공공 디지털 서비스로 재구성하는 것이다. 특히 지연증명, 실시간 열차 운행 정보, 유실물, 역사 편의시설, 냉난방 민원, 채용 및 공지 등 시민의 이용 목적이 분명한 기능을 상황 중심으로 묶어 접근성을 높이는 데 초점을 두었다.

## 2. 과제 요구사항 해석

| 구분 | 요구사항 해석 | 본 제출물 반영 |
|---|---|---|
| 문제 정의 | 현재 홈페이지를 시민 중심 관점에서 분석 | 사용자 문제와 핵심 Pain Point 정리 |
| Design Thinking | 공감, 문제정의, 아이디어, 프로토타입, 테스트 관점 적용 | 페르소나, 사용자 여정, 개선 방향 반영 |
| AX 기획 | 생성형 AI가 개입할 수 있는 서비스 포인트 설계 | AI 시민 이동 도우미, 민원 자동 분류, 쉬운 안내 설계 |
| PRD | 개발자가 이해 가능한 요구사항 문서 작성 | 기능/비기능/데이터/화면 요건 정리 |
| 프로토타입 | HTML/CSS/JS 또는 바이브코딩 도구로 구현 | 정적 웹 프로토타입 제작 |
| 제출 방식 | 완성 결과물을 URL로 제출 | GitHub Pages 배포 가능 구조 제공 |

## 3. 현재 홈페이지 문제 정의

서울교통공사 홈페이지는 공공기관으로서 많은 정보를 제공하지만, 시민 입장에서는 정보가 기능별·조직별로 분산되어 있어 원하는 업무를 빠르게 찾기 어렵다. 특히 지하철 이용 중 문제가 발생한 시민은 기관 소개보다 즉시 해결 가능한 기능을 먼저 필요로 한다.

### 핵심 문제

1. **기관 중심 정보 구조**
   - 메뉴가 기관의 업무 분류에 가깝고, 시민의 상황별 탐색 흐름과 다소 거리가 있다.
2. **즉시성 높은 기능의 접근성 부족**
   - 지연증명, 유실물, 열차 운행 정보, 냉난방 민원 등은 시민이 급하게 찾는 기능이므로 첫 화면에서 더 강하게 노출될 필요가 있다.
3. **디지털 약자와 외국인 이용자 고려 필요**
   - 고령자, 장애인, 외국인 관광객은 복잡한 메뉴명보다 쉬운 언어와 상황 중심 안내가 필요하다.
4. **채널 분산**
   - 공식 홈페이지, 또타24, 실시간 열차 정보, 채용 사이트 등이 분리되어 있어 통합 안내 경험이 약하다.
5. **AI 활용 가능성 미반영**
   - 생성형 AI를 통해 시민 질문을 이해하고, 공식 정보로 연결하는 서비스 경험을 설계할 수 있다.

## 4. Design Thinking 적용

### 4.1 Empathize: 핵심 사용자

| 사용자 | 대표 상황 | 주요 니즈 |
|---|---|---|
| 출근길 시민 | 지하철 지연으로 회사에 지각 | 지연증명서 빠른 발급 |
| 교통약자 | 엘리베이터, 화장실, 이동 동선 확인 | 역사 편의시설 정보 |
| 외국인 관광객 | 노선, 요금, 환승, 목적지 탐색 | 쉬운 다국어 안내 |
| 분실물 발생 시민 | 열차에 물건을 두고 내림 | 유실물센터와 신고 절차 |
| 지하철 이용 중 불편 민원인 | 냉난방, 혼잡, 안전 문제 발생 | 빠른 민원 접수와 처리 경로 |
| 취업 준비생 | 채용 공고 확인 | 채용 페이지 직접 접근 |

### 4.2 Define: 문제 정의문

시민은 지하철 이용 중 문제가 발생했을 때, 기관의 전체 조직 구조를 탐색하는 것이 아니라 “지금 내 상황을 해결할 수 있는 기능”을 10초 안에 찾고 싶어 한다. 따라서 홈페이지는 기관 소개 중심이 아니라 시민의 이동 상황과 문제 해결 흐름을 중심으로 재구성되어야 한다.

### 4.3 Ideate: 개선 아이디어

- 첫 화면에 “지금 필요한 업무” 중심의 빠른 해결 카드 배치
- 지연증명, 실시간 열차 정보, 유실물, 편의시설, 냉난방 민원 등을 최상단 노출
- AI 시민 이동 도우미를 통해 자연어 질문 기반 메뉴 추천
- 공공기관 정보는 별도 “공사 소개·홍보·채용” 영역으로 정리
- 모바일 우선 UI와 큰 버튼, 쉬운 문장 적용
- 공식 링크 연결 시 외부/내부 서비스 여부를 명확히 표시
- 정보 조회와 민원 접수 흐름을 분리하지 않고 상황별로 통합

### 4.4 Prototype: 구현 방향

본 프로토타입은 단일 페이지 정적 웹사이트로 구현하였다. 서버와 데이터베이스 없이도 GitHub Pages에 바로 배포할 수 있으며, 핵심 기능은 다음과 같다.

- 시민 상황별 빠른 링크 카드
- 검색어 기반 카드 필터링
- AI 도우미 시뮬레이션
- 공식 링크 모음
- PRD 요약 영역
- 접근성 고려 메뉴와 포커스 스타일

### 4.5 Test: 검증 기준

| 검증 항목 | 확인 기준 |
|---|---|
| 기능 접근성 | 지연증명, 실시간 정보, 유실물, 편의시설 메뉴가 첫 화면에서 보이는가 |
| 모바일 사용성 | 390px 모바일 화면에서도 카드와 버튼이 깨지지 않는가 |
| 키보드 접근성 | Tab, Enter, Esc로 주요 기능 사용이 가능한가 |
| 정보 신뢰성 | 공식 서울교통공사 관련 URL로 연결되는가 |
| 과제 적합성 | AX 기획, PRD, 프로토타입, URL 제출 구조를 모두 포함하는가 |

## 5. AX 기획안

### 5.1 AX 목표

생성형 AI를 단순 챗봇 장식으로 사용하는 것이 아니라, 시민의 질문을 이해하고 공식 메뉴로 연결하는 “공공서비스 탐색 보조자”로 활용한다.

### 5.2 AI 개입 포인트

| 영역 | AI 역할 | 기대 효과 |
|---|---|---|
| 메뉴 탐색 | 자연어 질문을 공식 메뉴로 매칭 | 메뉴 탐색 시간 단축 |
| 민원 분류 | 냉난방, 안전, 혼잡, 시설 민원 분류 | 접수 오류 감소 |
| 지연 안내 | 지연증명서 발급 경로 안내 | 반복 문의 감소 |
| 유실물 안내 | 분실 장소·시간 기반 신고 절차 안내 | 유실물센터 문의 효율화 |
| 교통약자 안내 | 편의시설 위치 정보로 연결 | 이동 편의 향상 |
| 다국어 안내 | 쉬운 한국어/영어 안내 문장 제공 | 외국인 접근성 향상 |

### 5.3 AI 응답 원칙

- 공식 홈페이지와 공공데이터에 근거한 정보만 안내
- 불확실한 정보는 단정하지 않고 공식 링크로 연결
- 개인정보, 결제정보, 민감정보는 수집하지 않음
- 민원 접수는 사용자가 확인 후 제출하도록 설계
- 응급·안전 상황은 챗봇보다 즉시 신고/역무원/고객센터 연결을 우선 안내

## 7. PRD

### 7.1 제품명

서울교통공사 시민 중심 AX 홈페이지 프로토타입

### 7.2 제품 목표

서울교통공사 홈페이지 이용자가 자주 찾는 기능을 상황 중심으로 재배치하고, AI 안내 흐름을 통해 시민이 필요한 정보를 더 빠르게 찾을 수 있도록 한다.

### 7.3 핵심 사용자

- 서울 지하철 일반 이용 시민
- 출퇴근·등하교 이용자
- 고령자·장애인 등 교통약자
- 외국인 관광객
- 민원 또는 분실물 발생 시민
- 채용 정보 탐색자

### 7.4 주요 기능 요구사항

| ID | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| F-01 | 빠른 링크 카드 | 자주 찾는 기능을 카드로 제공 | High |
| F-02 | 검색 필터 | 키워드 입력 시 관련 카드만 표시 | High |

## 9. GitHub Pages 배포 절차

1. GitHub에서 새 저장소를 생성한다.
2. 본 패키지의 파일을 저장소 루트에 업로드한다.
3. `Settings > Pages`로 이동한다.
4. Source를 `Deploy from a branch`로 선택한다.
5. Branch는 `main`, Folder는 `/root`로 선택 후 저장한다.
6. 배포 후 `https://사용자명.github.io/저장소명/` 형식의 URL을 제출한다.

GitHub Actions 방식으로 배포하려면 `.github/workflows/pages.yml` 파일을 함께 사용한다.

## 10. 생성형 AI 활용 고지

본 과제 수행 과정에서 생성형 AI는 요구사항 정리, 리포트 초안 작성, PRD 구조화, HTML/CSS/JS 프로토타입 코드 작성 보조에 활용되었다. 다만 최종 제출 전 사실 확인, 공식 링크 검증, 문구 수정, 제출 책임은 제출자에게 있다.

## 11. 한계와 후속 개선

본 프로토타입은 과제 제출용 정적 시제품이므로 실제 서울교통공사 시스템과 직접 연동하지 않는다. 후속 단계에서는 공식 API 기반 실시간 데이터 연동, 다국어 지원, 실제 AI 검색/RAG, 민원 접수 상태 조회, 교통약자 동선 안내를 추가할 수 있다.
