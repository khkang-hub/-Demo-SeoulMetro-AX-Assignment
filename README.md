# 서울교통공사 시민 중심 AX 홈페이지 리뉴얼

## 프로젝트명

서울교통공사 시민 중심 AX 홈페이지 리뉴얼

## 문제정의

서울교통공사 공식 홈페이지는 공공기관으로서 다양한 정보를 제공하고 있으나, 시민 입장에서는 실제 이용 상황에서 필요한 기능을 빠르게 찾기 어렵다는 문제가 있다. 지하철 이용자는 기관 소개나 조직 정보보다 실시간 열차 운행, 지연증명서, 유실물, 역사 편의시설, 운임, 냉난방 민원, 공지사항처럼 당장 필요한 정보를 우선적으로 찾는다.

현재 홈페이지는 여러 서비스가 메뉴별로 분산되어 있어 사용자가 원하는 기능을 찾기까지 탐색 단계가 길어질 수 있다. 특히 출근길 지연, 분실물 발생, 교통약자 이동, 외국인 이용, 열차 내 불편 민원처럼 즉시성이 높은 상황에서는 사용자가 복잡한 메뉴 구조를 이해하기보다 자신의 상황에 맞는 해결 경로를 바로 확인할 수 있어야 한다.

따라서 본 프로젝트는 서울교통공사 홈페이지를 기관 중심 정보 제공 화면이 아니라 시민의 이동 상황과 문제 해결 흐름을 중심으로 재구성하는 것을 목표로 한다.

## 핵심 사용자

### 1. 출퇴근·등하교 시민

출근길이나 등교 시간에 지하철을 자주 이용하는 시민이다. 실시간 열차 운행 정보, 지연 발생 여부, 간편지연증명서 발급 기능을 빠르게 확인해야 한다.

### 2. 유실물 발생 시민

열차 또는 역사에서 물건을 두고 내린 시민이다. 유실물센터 위치, 분실물 조회 방법, 문의 절차를 빠르게 알고 싶어 한다.

### 3. 교통약자 및 동반 이용자

장애인, 고령자, 영유아 동반 가족 등 이동 편의시설 정보가 필요한 사용자이다. 엘리베이터, 에스컬레이터, 화장실, 수유실, 환승 동선 등의 정보를 쉽게 찾을 수 있어야 한다.

### 4. 외국인 관광객

서울 지하철을 이용해 목적지로 이동하려는 외국인 사용자이다. 노선, 환승, 운임, 승차권, 실시간 운행 정보처럼 이동에 직접 필요한 정보를 단순하고 직관적인 구조로 제공받아야 한다.

### 5. 민원·불편 신고 사용자

열차 내 냉난방, 혼잡, 안전 문제, 시설 불편 등을 겪은 시민이다. 어떤 채널로 문의해야 하는지, 어떤 정보를 함께 입력해야 하는지 안내받을 필요가 있다.

### 6. 채용 및 공사 정보 탐색자

서울교통공사 채용정보, 공지사항, 자료실, 공사 소개 등 기관 정보를 확인하려는 사용자이다. 시민 서비스와 기관 정보를 구분해 명확하게 탐색할 수 있어야 한다.

## Pain Point

### 1. 원하는 기능까지의 탐색 단계가 길다

시민이 가장 자주 찾는 기능이 메인 화면에서 충분히 강조되지 않으면 여러 메뉴를 이동해야 한다. 특히 지연증명서, 유실물, 실시간 열차 정보처럼 긴급성이 있는 서비스는 첫 화면에서 바로 접근할 수 있어야 한다.

### 2. 기관 중심 메뉴와 시민 상황 사이에 간극이 있다

공공기관 홈페이지는 조직, 업무, 제도, 홍보 정보가 함께 제공되기 때문에 메뉴 구조가 기관 내부 분류에 가까워질 수 있다. 그러나 시민은 “내가 지금 무엇을 해결해야 하는가”를 기준으로 서비스를 찾는다.

### 3. 교통약자와 외국인에게 탐색 부담이 크다

복잡한 메뉴명, 작은 버튼, 여러 단계의 이동은 디지털 접근성이 낮은 사용자에게 불리하다. 교통약자와 외국인 이용자는 쉬운 문장, 큰 버튼, 직관적인 서비스 그룹이 필요하다.

### 4. 민원과 안내 채널이 분산되어 있다

또타24 챗봇, 실시간 열차 정보, 유실물, 지연증명, 공식 홈페이지 메뉴가 서로 분산되어 있어 사용자가 어떤 채널을 선택해야 할지 혼란을 겪을 수 있다.

### 5. AI 활용이 단순 장식으로 끝날 위험이 있다

AI 챗봇을 단순히 배치하는 것만으로는 시민 경험이 개선되지 않는다. 시민 질문을 이해하고 공식 서비스 경로로 연결하는 안내형 AX 구조가 필요하다.

## 리뉴얼 방향

### 1. 시민 상황 중심의 첫 화면 구성

메인 화면에서 “실시간 운행”, “자주 찾는 서비스”, “길찾기”, “AI 안내”, “민원·안전”을 바로 볼 수 있도록 구성했다. 기관 홍보보다 시민이 당장 해결해야 하는 업무를 우선 배치했다.

### 2. 자주 찾는 서비스 카드화

간편지연증명서, 실시간 열차 운행, 유실물센터, 역사 편의시설, 냉난방 요청, 운임·승차권, 공지사항, 채용정보를 카드 형태로 배치했다. 사용자는 메뉴명을 일일이 탐색하지 않고 자신의 목적에 맞는 카드를 선택할 수 있다.

### 3. 검색 기반 빠른 접근

메인 검색창에서 “지연증명”, “유실물”, “엘리베이터”, “냉난방”, “채용” 등 키워드를 입력하면 관련 시민 서비스 카드로 이동하도록 설계했다. 이는 메뉴 탐색이 어려운 사용자에게 빠른 접근 경로를 제공한다.

### 4. AI 시민 안내 도우미

AI 안내 영역은 사용자의 질문을 공식 서비스로 연결하는 역할을 한다. 예를 들어 “지하철이 지연돼 증명서가 필요해요”를 선택하면 간편지연증명서로, “열차에 물건을 두고 내렸어요”를 선택하면 유실물센터로 연결되는 방식이다.

### 5. 서울교통공사 정체성 반영

서울교통공사의 블루 계열 색상, 공공기관형 정돈된 레이아웃, 지하철 노선 색상 요소를 반영했다. 좌측 상단 CI 영역은 `ci.png` 이미지를 불러오도록 구현하여, 공식 홈페이지에서 캡처한 CI 이미지를 업로드하면 실제 CI로 교체될 수 있도록 했다.

### 6. 모바일 및 접근성 고려

버튼과 카드 크기를 충분히 확보하고, 모바일 화면에서는 메뉴가 접히도록 구성했다. 키보드 접근을 고려해 본문 바로가기, 포커스 표시, 명확한 링크 텍스트를 적용했다.

## AX 적용 포인트

### 1. 자연어 기반 시민 서비스 안내

사용자가 기관 메뉴명을 정확히 몰라도 “지연됐어요”, “물건을 잃어버렸어요”, “엘리베이터가 필요해요”처럼 일상적인 표현으로 필요한 서비스를 찾을 수 있도록 설계했다.

### 2. 민원 유형 자동 분류

냉난방, 안전 위험, 교통약자 이동, 유실물, 지연증명처럼 시민 문의가 자주 발생하는 유형을 사전에 분류하고, 각 유형에 맞는 공식 경로로 안내한다.

### 3. 공식 정보 기반 연결

AI가 임의로 답변을 생성하기보다 공식 홈페이지, 실시간 열차 정보, 또타24 챗봇, 유실물센터, 간편지연증명서 등 검증 가능한 경로로 연결하는 구조를 적용했다.

### 4. 반복 문의 감소

시민이 자주 문의하는 내용을 첫 화면과 AI 안내에서 먼저 해결하도록 하여 콜센터, 역무실, 현장 직원에게 집중되는 반복 문의를 줄일 수 있다.

### 5. 접근성 보조

디지털 사용이 익숙하지 않은 시민도 복잡한 메뉴 대신 질문형 안내와 카드형 버튼을 통해 서비스를 찾을 수 있다.

## 핵심 기능

### 1. 실시간 운행 안내

메인 우측 카드에서 실시간 열차 운행 정보를 바로 확인할 수 있도록 구성했다. 사용자는 열차 위치와 운행 상황을 확인하는 공식 페이지로 이동할 수 있다.

### 2. 자주 찾는 시민 서비스

다음 서비스를 카드형 UI로 제공한다.

- 간편지연증명서
- 실시간 열차 운행
- 유실물센터
- 역사 편의시설
- 냉난방 요청
- 운임·승차권
- 공지사항
- 채용정보

### 3. 메인 검색

메인 검색창에서 키워드를 입력하면 관련 서비스 카드 영역으로 이동한다. 사용자는 메뉴 구조를 몰라도 필요한 기능을 빠르게 찾을 수 있다.

### 4. 빠른 검색어 버튼

지연증명, 유실물, 엘리베이터, 냉난방, 채용 등 대표 키워드를 버튼으로 제공한다. 클릭 시 해당 키워드 기반으로 서비스 카드가 필터링된다.

### 5. 길찾기와 이용 준비

출발역과 도착역을 입력하면 사이버스테이션과 운임안내로 이동할 수 있는 안내를 제공한다. 실제 서비스 확장 시 경로 탐색 API와 연동할 수 있다.

### 6. AI 시민 안내 도우미

사용자가 자주 겪는 상황을 선택하면 그 상황에 맞는 안내 문구가 표시된다. 현재 프로토타입에서는 실제 AI API 호출 없이 사용자 흐름을 확인할 수 있는 시뮬레이션 형태로 구현했다.

### 7. 민원·안전·도움 요청

냉난방 불편, 안전 위험, 교통약자 이동 등 주요 민원 유형을 분류하여 안내한다. 시민이 어떤 정보를 준비해야 하는지 쉽게 이해할 수 있도록 구성했다.

### 8. 통합 서비스 메뉴

이동 정보, 문제 해결, 공사 정보로 주요 서비스를 다시 분류했다. 기존 기관 중심 메뉴보다 시민 목적 중심으로 재구성했다.

### 3. 화면 구성 요구사항

- 상단에는 서울교통공사 CI와 주요 메뉴를 배치한다.
- 메인 히어로 영역에는 시민 서비스 중심 메시지와 검색 기능을 배치한다.
- 실시간 운행 정보는 첫 화면 우측 카드로 제공한다.
- 자주 찾는 시민 서비스는 카드형 UI로 구성한다.
- 길찾기 영역은 출발역과 도착역 입력 후 공식 경로로 안내한다.
- AI 안내 영역은 시민 상황별 메시지를 표시한다.
- 민원·안전 영역은 냉난방, 안전 위험, 교통약자 이동을 분류한다.
- 전체 메뉴는 시민 목적별로 재구성한다.

### 4. 인터랙션 요구사항

- 검색 버튼 클릭 시 관련 서비스 카드 영역으로 이동한다.
- 빠른 검색어 버튼 클릭 시 서비스 카드가 필터링된다.
- AI 안내 버튼 클릭 시 사용자 메시지와 안내 메시지가 변경된다.
- 길찾기 버튼 클릭 시 입력한 출발역과 도착역을 포함한 안내 문구가 표시된다.
- 모바일 화면에서는 메뉴 버튼으로 내비게이션을 열고 닫을 수 있다.
- Esc 키를 누르면 모바일 메뉴가 닫힌다.

### 5. 배포 요구사항

- 외부 사용자가 별도 로그인 없이 접속할 수 있어야 한다.

## 미구현 기능 및 향후 개선 방향

### 1. 실제 API 연동

현재 프로토타입은 정적 웹페이지이므로 실제 열차 운행 데이터나 역사 편의시설 데이터를 API로 불러오지 않는다. 향후 서울교통공사 또는 공공데이터 API와 연동하면 실시간 정보 제공이 가능하다.

### 2. 실제 AI 챗봇 연동

현재 AI 시민 안내 도우미는 시뮬레이션 형태이다. 향후 공식 홈페이지 문서와 민원 안내 데이터를 기반으로 RAG 방식의 AI 안내 챗봇을 구현할 수 있다.

### 3. 다국어 서비스

외국인 관광객을 위해 영어, 중국어, 일본어 안내를 추가할 필요가 있다. 특히 운임, 환승, 승차권, 유실물 안내는 다국어 지원 효과가 크다.

### 4. 역사 편의시설 상세화

현재는 역사 편의시설 안내의 개념만 반영되어 있다. 향후 역별 엘리베이터, 에스컬레이터, 화장실, 수유실, 교통약자 동선 데이터를 상세하게 연결할 수 있다.

### 5. 민원 접수 상태 조회

현재는 민원 접수 경로 안내까지만 구현되어 있다. 실제 서비스에서는 민원 접수, 처리 상태 조회, 알림 기능까지 확장할 수 있다.

### 6. 접근성 고도화

색상 대비, 스크린리더 테스트, 키보드 전용 사용성 테스트, 고령자 대상 사용성 테스트를 추가로 수행할 필요가 있다.

## 기대효과

### 1. 시민의 정보 탐색 시간 단축

자주 찾는 서비스를 첫 화면에 배치하고 검색 기능을 제공함으로써 시민이 필요한 정보를 찾는 시간을 줄일 수 있다.

### 2. 현장 문의와 반복 민원 감소

지연증명, 유실물, 냉난방, 편의시설 등 반복 문의가 많은 기능을 온라인에서 먼저 안내하여 역무실, 콜센터, 현장 직원에게 집중되는 문의를 줄일 수 있다.

### 3. 공공서비스 접근성 향상

고령자, 교통약자, 외국인 등 다양한 사용자가 더 쉽게 정보를 찾을 수 있도록 카드형 UI와 쉬운 문장을 적용했다.

### 4. 서울교통공사 브랜드 신뢰도 강화

공식 CI와 서울교통공사의 블루 계열 색상, 정돈된 공공기관형 디자인을 반영해 신뢰감 있는 홈페이지 경험을 제공한다.

### 5. AX 기반 서비스 개선 가능성 제시

AI 시민 안내 도우미를 통해 생성형 AI가 공공기관 홈페이지에서 어떻게 활용될 수 있는지 보여준다. 단순 챗봇이 아니라 공식 정보 연결, 민원 유형 분류, 사용자 상황별 안내라는 실질적인 AX 적용 방향을 제안한다.

### 6. 향후 실제 개발을 위한 요구사항 기반 마련

프로토타입과 PRD를 함께 작성하여 실제 개발 단계에서 필요한 기능, 화면, 데이터, 비기능 요구사항을 구체적으로 검토할 수 있다.
